**Issue:** Seb not responding to dashboard chat messages

**Root Causes Found:**

1. **Bug #1 - Polling Property Name:**
   - ❌ AgentChat.tsx checked `latestMsg.text`
   - ✅ API returns `latestMsg.content`
   - **Fix:** Updated polling logic to use `.content`

2. **Bug #2 - Hardcoded Session:**
   - ❌ API looked for `agent:main:main` session (old: `0e033c6d`)
   - ✅ New active session created: `a5da7203-6d9f-4c1b-a7aa-ff3051d627f3`
   - **Fix:** Updated API to read latest `.jsonl` session file instead of hardcoded key

3. **Bug #3 - Agent Tool Loop:**
   - Seb got stuck in infinite tool call loop
   - Every tool call spawned more tool calls
   - **Fix:** Killed stuck processes, restarted fresh

**Files Modified:**
- `components/AgentChat.tsx` - Fixed polling property
- `app/api/openclaw/session/route.ts` - Use latest session file
