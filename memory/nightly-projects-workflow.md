# Nightly Projects Workflow 🌙

**Purpose:** Autonomous overnight development to build productivity tools, life improvements, and cool projects.

**Schedule:** Every night 23:00 - 07:00 GMT
**Morning Briefing:** 07:30 GMT daily

---

## Workflow Structure

### Phase 1: Ideation (23:00 - 00:00)
**Agent:** Research Coordinator
- Scan productivity/life-hack forums, GitHub trending, HN, Reddit
- Review USER.md for pain points and preferences
- Check recurring-patterns.md for automation opportunities
- Generate 5-10 raw ideas
- Score ideas by: impact, feasibility, Adam-fit
- Select top 3 for development

### Phase 2: Deep Dive (00:00 - 02:00)
**Agent:** Architecture Planner
- Iterate selected ideas until "great"
- Research existing solutions (avoid reinventing)
- Design technical architecture
- Create implementation plan with milestones
- Estimate resource needs

### Phase 3: Build (02:00 - 05:00)
**Agent:** Builder (spawn multiple if parallel tasks)
- Implement MVP
- Write tests
- Handle blockers (research workarounds, try alternatives)
- Document as we go

### Phase 4: Polish (05:00 - 06:00)
**Agent:** Quality Assurance
- Test end-to-end
- Fix bugs
- Improve UX
- Polish documentation

### Phase 5: Morning Prep (06:00 - 07:00)
**Agent:** Content Curator
- Gather headlines: World news, Financial markets, F1, Sports, Tech
- Summarize overnight project work
- Prepare delivery format

---

## Blocker Resolution Protocol

When blocked:
1. Try 3 different approaches immediately
2. Spawn research sub-agent to investigate
3. Search web for solutions
4. Check if alternative tool/library exists
5. If truly stuck, document blocker and pivot to next task

**Cost threshold:** Max 30 min per blocker before escalation/rotation

```
[Pattern noticed or idea for today]
```

1. **Productivity** — Workflow automation, time savers
2. **Data** — Visualizations, tracking, dashboards
3. **Content** — Generators, formatters, publishers
4. **Tools** — CLI utilities, shortcuts, integrations
5. **Fun** — Creative projects, experiments, art

- ✅ Deliver by 07:30 GMT
- ✅ Working MVP (not perfect, but functional)
- ✅ Documented how to use
- ✅ Tested at least once
- ✅ Overcame blockers or documented why not
